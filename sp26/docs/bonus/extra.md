## 类型

### 基础类型

!!! info "完成本部分内容最高可获得 1 分加分"

在基础实验中，我们只支持了 `int` 和 `void` 类型，你可以尝试支持更多的类型，你可以尝试着实现 `bool`、`char`、`float` 等类型，其中有些类型不是 8 位的，你需要考虑如何保存这些类型。也可以尝试着实现字符串字面量类型，但请注意我们的 Venus 和中间代码的 read/write 函数只支持整数，你可以查看 [RV32 工具链指南](../appendix/rv32_toolchain.md)，利用 Linux 的系统调用来完成字符串的输入输出。

你也可以考虑实现指针类型，或者像 Java 一样的引用类型。这样函数的返回值也能是 `int[]` 这样的类型。

常量类型也是一个很好的选择，你可以实现 `const` 关键字，来定义常量变量，常量变量的值不能被修改。常量相关的文法定义你可以直接参考 SysY 语音的原版文法。

### 类

!!! info "完成简单 C 结构体最高可获得 1 分加分，完成带函数的结构体最高可获得 1.5 分加分"

你可以尝试着实现类似 C++ 的类，或者简单的实现一个 C 中的结构体。例如：

```c
struct StructIdentifier {
    Type1 member1;
    Type2 member2;
} instance;
```

或更加复杂的带有函数的：

```c++
class ClassIdentifier {
    Type1 member1;
    Type2 member2;
    ClassIdentifier(/* some args */) {
        // constructor
    }
    ReturnType memberFunction(/* some args */) {
        // member function
    }
    ~ClassIdentifier() {
        // destructor
    }
} instance;
```

然后使用 `.` 或 `->` （如果你实现了指针）来访问成员变量和成员函数。

### 类型推断

!!! info "完成本部分内容最高可获得 0.25 分加分"

你可以尝试实现类型推断，即编译器可以根据上下文推断变量的类型，就像 C++ 中的 `auto` 关键字。例如：

```c
auto a = 1;     // a 的类型为 int
```

## 基础运算

!!! info "实现复杂嵌套表达式等可获得最高 0.5 分加分，实现更多运算符可获得最高 0.5 分加分"

通过我们定义的 CFG 生成的语言，只能支持一些简单的运算，存在着如下问题：

- 无法实现复杂嵌套的条件表达式，如 `(a || b) && c` 等。你可以修改文法，或使用优先级来解决。
- 未定义的行为，如 `1 < 2 < 3`。你可以引入 `bool` 变量，或者严格的类型检查来解决。

除此之外，你可以尝试着实现更多的运算符，但请注意优先级，例如：

- 位运算符：`&`、`|`、`^`、`~`、`<<`、`>>`
- 自增运算符：`++`、`--`
- 三目运算符：`? :`（注意条件表达式可能存在的短路问题）
- 逗号表达式：`,`
- etc.

## 语句

### 循环语句

!!! info "完成本部分内容最高可获得 0.5 分加分"

在基础实验中，我们只实现了 while 循环，你可以尝试实现更多的循环语句，如 `for` 循环，`do while` 循环等。例如，`for` 循环的定义如下：

```c
for (<init>; <condition>; <increment>) <block or statement>
```

你可以实现逗号表达式来实现更加复杂的初始化、条件增量语句，也可以实现在初始化时允许声明新变量等。请你注意在进入 `for` 语句时的作用域问题。

### 跳转语句

!!! info "完成本部分内容最高可获得 0.25 分加分"

在基础实验中，我们只实现了 `return` 语句。对于循环控制中的 `break`、`continue` 语句，你可以尝试实现。请注意，`break`、`continue` 语句仅在循环中有效，不在循环中则为语法错误，且只能作用于所在的循环层次。

## 函数

### 函数重载

!!! info "完成本部分内容最高可获得 0.5 分加分"

你可以尝试实现函数重载，即允许多个函数拥有相同的函数名，但参数列表不同。例如：

```c
int add(int a, int b) {
    return a + b;
}

int add(int a, int b, int c) {
    return a + b + c;
}
```

在调用时，编译器会根据参数列表选择正确的函数。

在实现上，你可以将参数列表不同的函数视为不同的函数，并对每个函数给一个唯一的标识符，将代码中对同名函数的调用替换为对不同标识符的调用即可。比如，可以在建立符号表时将两个 `add` 函数分别标记为 `add_int_int` 和 `add_int_int_int`。在遇到对 `add` 函数的调用时，需要匹配调用处的参数的个数以及每个参数的类型，寻找对应的重载函数。例如，对于 `add(1, 2)`，会发现它有两个 `int` 类型的参数，因此会调用 `add_int_int` 函数。

### 嵌套函数

!!! info "完成本部分内容最高可获得 1.5 分加分"

你可以尝试实现嵌套函数，即在函数内部定义函数。这样的函数只能在定义它的函数内部调用，不能在外部调用。我们这里看一个例子：

```c
int outer() {
    int a = 1;
    int middle1() {
        return a;
    }
    int middle2() {
        int b = 2;
        int inner() {
            int c = middle1();
            return a + b + c;
        }
        return inner();
    }
    return middle2();
}
```

??? example "该例子的正确 C++ 写法"
    C++ 中不支持在函数内部定义函数，但可以使用 lambda 表达式来实现类似的功能：

    ```cpp
    int outer() {
        int a = 1;
        auto middle1 = [&]() {
            return a;
        };
        auto middle2 = [&]() {
            int b = 2;
            auto inner = [&]() {
                int c = middle1();
                return a + b + c;
            };
            return inner();
        };
        return middle2();
    }
    ```

=== "针对 RV32 工具链的编译器"

    你的编译器 compiler 必须支持**两个**命令行参数的情形，即：

    ```console
    $ ./compiler <input_file> <output_file>
    ```

    该程序必须接受一个输入的源代码文件名作为参数，并且将输出的汇编代码写入到指定的 `output_file` 文件中，我们只会使用这种方式来测试你的编译器。

    在提交的时候，你需要将你的编译器放到 `bonus2` 分支中，并且在 `config.toml` 中修改以下选项：

    ```toml
    use_qemu = true
    ```

    这样，我们的 CI 测试脚本在测试 Bonus 2 时会使用 RISC-V 32 工具链进行编译、运行，而不是直接使用 Venus 模拟器来运行。你可以运行以下命令来测试你的编译器：

    ```console
    $ python3 sp26-tests/test.py bonus2 .
    ```
    
=== "针对 Venus 模拟汇编的编译器"

    你的编译器 compiler 必须支持**三个**命令行参数的情形，即：

    ```console
    $ ./compiler <input_file> <output_file> --venus
    ```

    该程序必须接受一个输入的源代码文件名作为参数，并且将输出的汇编代码写入到指定的 `output_file` 文件中，第三个参数 `--venus` 是提示程序需要使用 Venus 测试。我们只会使用这种方式来测试你的编译器。

    运行以下命令来测试你的编译器：

    ```console
    $ python3 sp26-tests/test.py bonus2 .
    ```

在本次实验的测试脚本中，我们也会使用你放在 `appends/bonus2` 文件夹下的测试用例来测试你的编译器。
